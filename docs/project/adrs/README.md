## ADR Workflow

- create an ADR when a decision materially changes implementation, operations, privacy, or hardware tradeoffs
- use consecutive numbering
- record status as `Proposed`, `Accepted`, `Superseded`, or `Rejected`
- link the ADR from affected docs and execution milestones
